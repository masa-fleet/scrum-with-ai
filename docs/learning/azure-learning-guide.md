# Azure 学習ガイド（優先度付き）

対象: IT プロジェクトのエンジニア（開発、運用、DevOps）

## 優先度 P0（まずここ）

- [クラウド インフラストラクチャの概要: クラウドの概念について説明する](https://learn.microsoft.com/ja-jp/training/paths/microsoft-azure-fundamentals-describe-cloud-concepts/)
  - 種別: ラーニング パス
  - 目的: クラウドの基本概念、責任共有、クラウドサービスの種類を理解する
- [クラウド インフラストラクチャの概要: Azure の管理とガバナンスについて説明する](https://learn.microsoft.com/ja-jp/training/paths/describe-azure-management-governance/)
  - 種別: ラーニング パス
  - 目的: コスト管理、ガバナンス、監視、運用管理の基礎を理解する
- [クラウド インフラストラクチャの概要: ガイド付きプロジェクトに Azure スキルを適用する](https://learn.microsoft.com/ja-jp/training/paths/introduction-cloud-infrastructure-apply-azure-skills-guided-projects/)
  - 種別: ラーニング パス
  - 目的: Blob、Functions、RBAC、コストガードレールなどを手を動かして学ぶ

## 優先度 P1（エンジニア向け Azure AI）

- [AzureでAIアプリケーションとエージェントを始める](https://learn.microsoft.com/ja-jp/training/paths/get-started-ai-apps-agents/)
  - 種別: ラーニング パス
  - 目的: Azure AI と Foundry の基礎から、AI アプリとエージェント開発の入口を学ぶ
- [Microsoft Foundry で生成 AI アプリを開発する](https://learn.microsoft.com/ja-jp/training/paths/develop-generative-ai-apps/)
  - 種別: ラーニング パス
  - 目的: Foundry を使った生成 AI アプリの実装フローを学ぶ

## 優先度 P1（最新情報の追跡）

- [Microsoft Build](https://build.microsoft.com/)
  - 開催時期: 毎年 5〜6 月ごろ（年により前後）
  - 使い方: オンデマンド視聴、次回通知登録、発表後の関連リソース確認
- [Microsoft Ignite](https://ignite.microsoft.com/)
  - 開催時期: 毎年 10〜11 月ごろ（年により前後）
  - 使い方: オンデマンド視聴、アジェンダ確認、ハンズオンとセッションで運用系トピックを追跡

## 学習順（最小構成）

1. P0 の 3 つの基礎ラーニング パスを完了
2. P1 は「AzureでAIアプリケーションとエージェントを始める」→「Foundry で生成 AI アプリを開発する」の順で進める
3. Build / Ignite（P1）を必要に応じて確認して最新情報を補完

## 運用ルール

- 新しいリンク追加時は Microsoft 公式を優先する
- 1 つ追加したら 1 つ削除を検討し、情報量を増やし過ぎない
- 優先度は P0/P1 の 2 段階のみを維持する
