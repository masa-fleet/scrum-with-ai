# コントリビューションガイド

## ポリシー

- **AI がコード・ドキュメントを作成。** 人間はレビューと承認のみ担当する。
- Copilot cloud agent に指示して変更を生成する。
- [Expand your team with Copilot](https://github.com/skills/expand-your-team-with-copilot) を参照。

## コントリビューションの範囲

- **Microsoft 社内メンバー:** Issue 登録・レビュー・承認、Copilot cloud agent への指示出し
- **Microsoft 社外の方:** Issue 登録のみ

## 貢献の流れ（Microsoft 社内メンバー向け）

1. Issue 作成（目的・要件を記載）
2. Copilot cloud agent に割り当て
3. PR をレビュー・承認・マージ

## ブランチ・プルリクエストのルール

- `main` ブランチへの直接プッシュは禁止。必ずプルリクエストを経由する。
- プルリクエストには対応する Issue を紐付ける。
- レビュアーは最低 1 名以上設定する。

## Issue 登録のガイドライン（全員共通）

- 重複を確認する。
- タイトルは明確に記載する。
- 背景・目的・受け入れ基準を記載する。

## 行動規範

- [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/) に従う。
